## 🐛 Troubleshooting

### Issue: "GEMINI_API_KEY not configured"
**Solution:**
1. Check ai-services/.env has GEMINI_API_KEY
2. Restart Python service
3. Test: `curl http://localhost:8001/ai/health`

### Issue: "Connection refused" on port 8001
**Solution:**
1. AI service not running
2. Run: `cd ai-services && python main.py`
3. Check no firewall blocking 8001

### Issue: "0 questions generated"
**Solution:**
1. Check AI service logs
2. Verify GEMINI_API_KEY is valid
3. Try fallback: Still shows fallback questions

### Issue: No MongoDB documents
**Solution:**
1. Check MongoDB is running
2. Check user_id is correct
3. Questions might not be stored yet (5-10 sec delay)
