# LLM Question Generation Fix - 500 Error Resolution

## Problem
When users clicked on questions generated by LLM and tried to start a practice session, they got a 500 error:
```
Question not found: contains-duplicate. Cannot create session without a valid wrapped problem.
```

## Root Cause Analysis
The issue was a mismatch in problemIds between what the frontend received and what was saved to the database:

1. **LLM generates** questions with titles like "Contains Duplicate"
2. **saveWrappedQuestionsToDatabase()** would:
   - Convert to wrapped format (schemaVersion: 2)
   - Generate new problemIds: `llm-{slug}-{timestamp}`
   - Save to QuestionBank
3. **enrichWithQuestionBankLinks()** would:
   - Return the original `processedQuestions` (not the wrapped versions)
   - These don't have the wrapped problemIds
4. **Frontend** sends original problemId (e.g., "contains-duplicate")
5. **Backend** tries to find it in QuestionBank but fails because the saved problemId was different

## Changes Made

### 1. **Improved LLM Question Return Logic** 
[backend/src/services/llmQuestionGenerationService.js](backend/src/services/llmQuestionGenerationService.js) - Lines 198-232

**Before:**
- Saved wrapped questions to database
- Returned original processed questions (with mismatched problemIds)

**After:**
- Saved wrapped questions to database
- Returns the saved wrapped questions with correct problemIds
- Fallback to enriched processing if save fails

```javascript
const savedWrappedQuestions = await this.saveWrappedQuestionsToDatabase(wrappedQuestions);

// Return saved wrapped questions (with correct problemIds)
const questionsToReturn = savedWrappedQuestions.length > 0 ? 
  savedWrappedQuestions.map(q => ({
    problemId: q.problemId,  // ✅ CORRECT: From wrapped version
    problemTitle: q.title,
    ...
  })) :
  await this.enrichWithQuestionBankLinks(processedQuestions);
```

### 2. **Better Error Handling in saveWrappedQuestionsToDatabase()**
[backend/src/services/llmQuestionGenerationService.js](backend/src/services/llmQuestionGenerationService.js) - Lines 972-1024

**Improvements:**
- Force schemaVersion: 2 on all saved questions
- Force isActive: true for new questions
- Update legacy questions if they already exist but aren't v2
- Better error logging with validation details
- Track which questions were successfully saved

### 3. **Improved Error Diagnostics in startSession()**
[backend/src/controllers/practiceController.js](backend/src/controllers/practiceController.js) - Lines 579-616

**Improvements:**
- Debug query to check if question exists but doesn't match v2 filter
- Detailed logging when question exists but fails validation
- Better error messages for v1 questions
- Helps identify whether question is missing vs. just not matching filter
